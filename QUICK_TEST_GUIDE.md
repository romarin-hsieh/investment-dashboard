# YFinance Metadata 測試頁面 - 快速使用指南

## 🚀 快速開始

### 1. 開啟測試頁面
```bash
# 方法 1: 啟動開發伺服器 (推薦)
npm run dev
# 然後訪問: http://localhost:5173/test-yfinance-metadata.html

# 方法 2: 直接開啟 (可能有 CORS 限制)
open test-yfinance-metadata.html
```

### 2. 基本測試流程 (5分鐘)

#### 步驟 1: 檢查初始狀態
1. 點擊 **「檢查狀態」**
2. 查看快取狀態 (首次使用應該是 ❌)

#### 步驟 2: 測試 API 連接
1. **預設測試**: 點擊 **「測試單個股票」** (測試 CRM)
2. **自訂測試**: 在輸入欄位輸入股票代號 (如 AAPL)，按 Enter 或點擊測試
3. **快速選擇**: 點擊 **「科技股」** → **「批量測試自訂股票」**
4. 觀察日誌，應該看到：
   ```
   [時間] ✅ AAPL: Technology - Consumer Electronics
   ```

#### 步驟 3: 測試批量處理
1. **預設批量**: 點擊 **「測試批量股票」** (測試 5 支預設股票)
2. **自訂批量**: 在批量輸入欄位輸入 "AAPL,MSFT,GOOGL"，點擊測試
3. **快速批量**: 點擊 **「全部24支股票」** → **「批量測試自訂股票」**
4. 等待處理完成，查看右側統計資訊更新

#### 步驟 4: 測試快取機制
1. 點擊 **「強制更新」**
2. 重新點擊 **「檢查狀態」**
3. 確認快取狀態變為 ✅

## ⚠️ 常見問題

### 問題 1: API 請求失敗
**症狀**: 日誌顯示 ❌ 所有代理失敗
**解決**: 
- 檢查網路連接
- 等待幾分鐘後重試 (可能是 API 限制)
- 使用開發伺服器而非直接開啟 HTML

### 問題 2: 頁面空白
**症狀**: 頁面無法載入或顯示空白
**解決**: 
- 使用現代瀏覽器 (Chrome, Firefox)
- 透過 `npm run dev` 啟動開發伺服器訪問

### 問題 3: 快取無法保存
**症狀**: 重新整理後快取消失
**解決**: 
- 確認瀏覽器允許 localStorage
- 不要使用隱私/無痕模式
