🔧 MFI Volume Profile Height Fix Debug

解決 chart-container 高度被壓縮的問題

🚨 問題分析

發現的問題：
  • chart-container 設定 style="height: 600px;"
  • 但實際只有 345.48px 的空間
  • 50個 bins × 12px = 600px 應該要的高度
  • 實際高度被 CSS flex 屬性壓縮了

🔍 根本原因

CSS Flex 佈局問題：
  • .volume-profile-chart 使用 flex: 1
  • .chart-container 使用 flex: 1
  • 父容器 .panel-content 有 height: 100% 限制
  • 這些設定讓容器與其他元素共享空間，而不是使用設定的固定高度

✅ 修正方案

已實施的修正：
  1. 移除 flex: 1 限制
    /* 修正前 */ .volume-profile-chart { flex: 1; /* ❌ 這會壓縮高度 */ } /* 修正後 */ .volume-profile-chart { flex-shrink: 0; /* ✅ 防止被壓縮 */ }
  2. 修正 chart-container
    /* 修正前 */ .chart-container { flex: 1; /* ❌ 這會壓縮高度 */ } /* 修正後 */ .chart-container { flex-shrink: 0; /* ✅ 使用設定的固定高度 */ }
  3. 移除父容器高度限制
    /* 修正前 */ .panel-content { height: 100%; /* ❌ 限制子元素展開 */ } /* 修正後 */ .panel-content { /* ✅ 讓內容自然展開 */ }

📊 預期結果

修正後的行為：
  • chart-container 將使用完整的 600px 高度
  • 50個 bins × 12px = 600px 完全展開
  • 不再被其他元素壓縮空間
  • 使用者可以看到完整的 Volume Profile
  • 響應式設計仍然正常工作

🧪 測試驗證

驗證步驟：
  1. 部署修正後的代碼
  2. 打開任一 StockDetail 頁面
  3. 檢查 MFI Volume Profile 區塊
  4. 使用開發者工具檢查 .chart-container 的實際高度
  5. 確認高度為 600px（50 bins × 12px）
  6. 確認所有 50 個 bins 都可見，無需滾動

🔧 技術細節

CSS 佈局策略變更：
  • 從 Flex 自適應 → 固定高度模式
  • 從空間共享 → 內容驅動高度
  • 從容器限制 → 自然展開
  • 保持響應式設計和其他功能不變
