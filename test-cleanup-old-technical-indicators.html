🧹 30天數據保留政策測試

測試技術指標文件清理功能，防止 repository 無限膨脹

📋 功能概述

目標: 只保留最近 30 天的技術指標日檔，其餘從 repo 刪除

清理範圍:

  • ✅ public/data/technical-indicators/YYYY-MM-DD_SYMBOL.json
  • ❌ public/data/technical-indicators/latest_index.json (永遠保留)
  • ❌ public/data/ohlcv/* (這次不動)
  • ❌ 任何非日檔命名的 JSON

🖥️ 本地測試命令

在執行前，請先在本地測試以確保安全：

1. Dry Run 測試 (不會真的刪除)

node scripts/cleanup-old-technical-indicators.js --dry-run --retention-days=30 --verbose

2. 指定日期測試 (可重現結果)

node scripts/cleanup-old-technical-indicators.js --dry-run --as-of=2025-12-29 --retention-days=30 --verbose

3. 實際執行 (小心使用)

node scripts/cleanup-old-technical-indicators.js --retention-days=30 --verbose

🎯 預期效果

執行後應該看到：
• 只保留最近 30 天的 YYYY-MM-DD_SYMBOL.json 文件
• latest_index.json 永遠保留
• 清理日誌顯示刪除的文件數量
• Repository 大小不再線性增長
安全檢查：
• 如果要刪除的文件 > 500 個，腳本會拒絕執行
• 如果 asOfDate 比最新文件日期新 > 7 天，腳本會拒絕執行
• 支援 --dry-run 模式，可以先預覽不實際刪除
